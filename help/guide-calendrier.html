📅 Guide Complet : Gestion des Calendriers

Cette documentation est votre guide de référence complet pour maîtriser la gestion des calendriers dans l'application. Elle couvre tous les aspects, des concepts de base aux cas complexes.

⚠️ Configuration requise : Langue du navigateur

Pour afficher les horaires au format 24h (HH:MM) dans cette application :
La langue de votre navigateur doit être réglée sur FRANÇAIS

  • Si vous voyez AM/PM : Changez la langue du navigateur en FRANÇAIS
  • Chrome : Paramètres → Langues → Sélectionnez FRANÇAIS
  • Firefox : Paramètres → Général → Langue → FRANÇAIS
  • Safari : Préférences Système → Langue et région → FRANÇAIS
  • Edge : Paramètres → Langues → FRANÇAIS
🚨 Important : Usage d'un serveur HTTP

Cette application **ne peut pas** être ouverte directement comme un fichier local (double-clic). Elle doit être exécutée via un **serveur Web local** (ex: Python, Node.js) pour que toutes les fonctionnalités soient actives.

1. Comprendre les calendriers

Qu'est-ce qu'un calendrier ?

Un calendrier est une configuration qui définit les horaires d'ouverture et fermeture d'un service sur une période donnée. Il détermine quand vos files d'attente sont accessibles, quand les appels sont traités, et quand le service est fermé.

Types de calendriers disponibles

Type Objectif Cas d'usage
Calendrier Vocal Gère les heures d'accès au système vocal (IVR) Support client, services d'information téléphonique
Calendrier Distribution Définit les horaires de distribution des appels entre files Routage intelligent, load balancing
Calendrier Cible Horaires spécifiques d'une destination ou queue Équipes distantes, zones géographiques différentes
À retenir : Les calendriers Vocal et Distribution travaillent ensemble. Le calendrier Vocal détermine si le service global est ouvert, tandis que le calendrier Distribution affine le routage des appels quand le service est actif.

2. Système de couleurs et identification visuelle

Comprendre les couleurs du calendrier

Chaque couleur représente un état du service pour un jour donné. Ce code couleur permet une identification rapide et visuelle de la configuration d'une journée.

Ouvert Service en service normal, file active
Ouvert (Sans attente) Service accessible, mais pas de mise en attente possible
Pré-fermeture Service en phase de fermeture, derniers appels acceptés
Fermé Service indisponible
Fermeture Hebdomadaire Définition en Semaine Type : jour sans service regular
Jour Exceptionnel Exception personnalisée pour une date spécifique
Jour Férié Jour officiel sans service

Identification des jours exceptionnels

Les jours marqués d'une bordure pleine (cadre épais) sur le calendrier annuel indiquent qu'une exception personnalisée a été définie pour ce jour. Cette exception a une priorité maximale et surpassera les horaires réguliers.

Exemple visuel : Un jour avec une bordure pleine et une couleur marron (🟫) indique une journée exceptionnelle définie précisément pour ce jour.

3. Hiérarchie de priorité (Comment le système choisit)

Lorsque plusieurs règles s'appliquent à un même jour, le système suit une hiérarchie stricte de priorités. Comprendre cette hiérarchie est essentiel pour configurer correctement vos calendriers.

1️⃣
Priorité MAXIMALE : Exception personnalisée
Si une exception est définie pour une date spécifique, elle l'emporte toujours.
2️⃣
Priorité ÉLEVÉE : Fermeture hebdomadaire
Les jours fermés définis dans la Semaine Type surpassent d'autres configurations.
3️⃣
Priorité PAR DÉFAUT : Semaine Type
Les horaires réguliers d'une semaine type s'appliquent si aucune exception n'existe.
Règle métier clé : Si vous fermez un jour dans la Semaine Type (par exemple le dimanche), cette fermeture sera effective pour chaque dimanche de l'année, SAUF si vous définez une exception pour un dimanche en particulier.

4. Créer un nouveau calendrier (Pas à pas)

Quand créer un nouveau calendrier ?

  • Vous lancez un nouveau service client
  • Vous créez une nouvelle file d'attente avec des horaires différents
  • Vous gérez une nouvelle zone géographique avec un fuseau horaire différent
  • Vous avez besoin de configurations d'horaires radicalement différentes

Procédure complète

  1. Accédez à la page de gestion des calendriers
    • Depuis le menu principal, choisissez le type de calendrier que vous souhaitez créer en cliquant sur:
      • Calendrier Vocal
      Calendrier De distribution
      Calendrier Cible

    • Choisissez le type : Vocal, Distribution ou Cible
  2. Cliquez sur le bouton "Nouveau calendrier"
    • Localisez le bouton 📆 Nouveau calendrier
  3. Remplissez l'identifiant (ID) du calendrier
    • Format obligatoire : SERVICEXXXXXX ou SERVICE_XXXXXX
    • Caractères autorisés : Lettres (A-Z), chiffres (0-9), tiret sans espace
    • Caractères interdits : Espaces, accents, caractères spéciaux
    • Astuce : Utilisez des noms métier compréhensibles (ex: SUPPORT_CLIENT_N1, VENTE_STANDARD)
    • Attention : Cet ID est unique et ne peut pas être changé après création
  4. Remplissez le nom du calendrier
    • Format : Texte libre, clair et descriptif
    • Exemple bon : "Support Technique - Équipe N1 - Europe"
    • Exemple mauvais : "Cal1", "xxx"
    • Ce nom s'affichera dans les listes et rapports
  5. Sélectionnez le fuseau horaire
    • Impact critique : Le fuseau horaire détermine comment les horaires absolus sont calculés
    • Exemple : Si vous mettez "09:00 CET", il sera 09:00 dans l'Europe Centrale
    • Erreur courante : Sélectionner le mauvais fuseau horaire décale tous les horaires
    • Si vous gérez plusieurs zones, créez des calendriers séparés par zone
  6. Validez la création
    • Cliquez sur Créer le calendrier
    • Attendre la confirmation (un message de succès apparaît)
  7. Définissez le type de calendrier
    • Vocal : Pour les calendriers contrôlant les messages audios et horaires d'ouverture
    • Distribution : Pour les calendriers contrôlant le routage vers les files d'attente (Queues)
  8. Configurez les horaires
    • Le calendrier est créé mais vide. Vous devez maintenant définir la Semaine Type et les exceptions
    • Voir section 5 et 6 ci-dessous
Conseil : Dupliquez un calendrier existant plutôt que d'en créer un nouveau si vous avez besoin d'horaires similaires. Cela vous fera gagner du temps significativement.

5. Éditer un calendrier existant

Accéder à l'éditeur de calendrier

  1. Sélectionnez le calendrier dans la liste des calendriers disponibles
  2. Cliquez sur ✏️ Éditer
  3. L'éditeur de calendrier s'ouvre dans une interface dédiée

Interface d'édition

L'éditeur comporte plusieurs sections tab(onglets) et éléments :

  • Vue annuelle du calendrier : Affiche les 12 mois de l'année avec codes couleurs
  • Onglet "Semaine Type" : Configuration des horaires réguliers (voir section 6)
  • Onglet "Jours d'exception" : Gestion des jours spéciaux et fériés (voir section 7)
  • Bouton "Sauvegarder" : Persiste vos changements (ESSENTIEL)
⚠️ Attention très importante : Les modifications ne sont sauvegardées qu'après avoir cliqué sur le bouton Sauvegarder. Si vous fermez l'éditeur sans sauvegarder, tous vos changements seront perdus.

6. Configurer la Semaine Type

Qu'est-ce que la Semaine Type ?

La Semaine Type est un modèle qui définit les horaires réguliers pour chaque jour de la semaine (Lundi au Dimanche). Cette configuration s'applique à chaque semaine de l'année SAUF quand une exception spécifique est définie pour un jour donné.

En d'autres termes : la Semaine Type est votre configuration "par défaut" pour l'ensemble de l'année.

Étapes de configuration

  1. Ouvrez l'onglet "Semaine Type"
    • Depuis l'éditeur de calendrier, localisez l'onglet Semaine Type
  2. Pour chaque jour de la semaine, choisissez un état
    • Fermé tous la journée : Le jour n'est pas en service
    • Ouvert avec horaires spécifiques : Définissez heure d'ouverture et fermeture
    • États avancés : "Sans attente" ou "Pré-fermeture" si configuré
  3. Définissez les horaires (si applicable)
    • Heure d'ouverture : Quand le service commence (ex: 09:00)
    • Heure de fermeture : Quand le service se termine (ex: 18:00)
    • Format : HH:MM (24h), ex: 14:30, 09:00
    • Validation : L'heure d'ouverture DOIT être avant l'heure de fermeture
  4. Répétez pour les 7 jours
    • Lundi, Mardi, Mercredi, Jeudi, Vendredi, Samedi, Dimanche
  5. Cliquez sur "Sauvegarder"
    • Vos changements sont validés et persistés

Exemple de configuration standard

Jour État Horaires
Lundi Ouvert 09:00 - 18:00
Mardi Ouvert 09:00 - 18:00
Mercredi Ouvert 09:00 - 18:00
Jeudi Ouvert 09:00 - 18:00
Vendredi Ouvert 09:00 - 17:00
Samedi Fermé -
Dimanche Fermé -
Ce que signifie cette configuration : Le service ferme en semaine du lundi au vendredi (fermé samedi et dimanche). Ouvert de 09:00 à 18:00 du lundi au jeudi, et ferme une heure plus tôt le vendredi (17:00).

7. Gérer les jours d'exception (Fériés et cas spéciaux)

Qu'est-ce qu'une exception ?

Une exception est une formation spécifique pour un jour particulier qui surpasse la Semaine Type. Elle vous permet de gérer :

  • Les jours fériés (1er janvier, 25 décembre, etc.)
  • Les jours de fermeture exceptionnelle (grève, maintenance, événement)
  • Les jours d'horaires spéciaux (réduction horaire, augmentation)
  • Les jours de fermeture uniquement en matinée ou en après-midi

Ajouter une exception

  1. Ouvrez l'onglet "Jours d'exception"
  2. Cliquez sur "Ajouter une exception"
    • Un formulaire s'affiche
  3. Sélectionnez la date
    • Utilisez le calendrier ou entrez la date manuellement
    • Format : JJ/MM/YYYY ou sélection visuelle
  4. Choisissez l'état pour ce jour
    • Fermé toute la journée : Par ex pour un jour férié
    • Ouvert avec horaires différents : Ex: 10:00 - 16:00 pour une réduction
    • État spécialisé : "Sans attente", "Pré-fermeture"
  5. Entrez une description (optionnel mais recommandé)
    • Ex: "Jour férié - Noël", "Fermeture maintenance", "Réunion d'équipe"
    • Cela aide à comprendre pourquoi cette exception existe
  6. Cliquez sur "Ajouter"
  7. Le jour apparaît maintenant dans la liste des exceptions
    • Sur le calendrier annuel, il est marqué avec une bordure et une couleur particulière
  8. Cliquez sur "Sauvegarder" pour persister

Modifier une exception existante

  1. Dans la liste des exceptions, identifiez le jour à modifier
  2. Cliquez sur l'icône ✏️ (éditer)
  3. Modifiez les détails (date, état, horaires, description)
  4. Cliquez sur "Mettre à jour"
  5. Sauvegardez les changements globaux

Supprimer une exception

  1. Dans la liste des exceptions, identifiez le jour à supprimer
  2. Cliquez sur l'icône 🗑️ (supprimer)
  3. Confirmez la suppression
  4. Sauvegardez
Exemple 1 - Jour férié :
  • Date : 25/12/2026 (Noël)
  • État : Fermé
  • Description : "Jour férié - Noel"
Exemple 2 - Horaires spéciaux :
  • Date : 24/12/2026 (Veille de Noël)
  • État : Ouvert
  • Horaires : 10:00 - 14:00 (horaires réduits)
  • Description : "Horaires spéciaux - Veille de Noël"

8. États et horaires : Guide détaillé

Les différents états disponibles

1. OUVERT (🟪 Lavande/Violet clair)

Signification : Le service est pleinement opérationnel. Les appels sont acceptés, traités immédiatement ou mis en attente normalement, les agents reçoivent les appels.

Quand l'utiliser : Heures de service normales et prévues.

Horaires : HH:MM - HH:MM (obligatoire)

2. OUVERT - SANS ATTENTE (🟪 Violet/Mauve)

Signification : Le service est accessible, mais la mise en attente des appels est désactivée. Les appels arrivent directement aux agents disponibles. Si aucun agent n'est libre, l'appel peut être rejeté ou redirigé.

Quand l'utiliser :

  • Fin de journée quand on réduit progressivement l'activité
  • Équipe en nombre réduit
  • Services de faible priorité ou secondaires

3. PRÉ-FERMETURE (🟫 Marron)

Signification : Phase de transition avant fermeture. Le service accepte encore des appels, mais on signale qu'il ferme bientôt. Peut décourager les appels ou orienter vers d'autres files.

Quand l'utiliser : 30-60 minutes avant la fermeture réelle.

4. FERMÉ (🟩 Gris foncé)

Signification : Le service est complètement fermé. Les appels sont rejetés ou redirigés vers une alternative (message d'accueil, autre file, boîte vocale).

Quand l'utiliser : Heures de fermeture (soirée, nuit, week-end, jours fériés).

Configuration d'une journée : Workflow typique

09:00
🡪
OUVERT
17:30
🡪
PRÉ-FERMETURE
18:00
🡪
FERMÉ
Interprétation : Le service ouvre à 09:00 en mode normal. À partir de 17:30, il passe en pré-fermeture (phase de transition). À 18:00, il ferme complètement.

9. Dupliquer un calendrier existant

Pourquoi dupliquer ?

Dupliquer est la façon la plus rapide de créer des calendriers similaires. Au lieu de reconfigurer chaque jour, vous copiez un calendrier existant et le modifiez légèrement.

Procédure

  1. Sélectionnez le calendrier à dupliquer
    • Choisissez parmi la liste des calendriers disponibles
  2. Cliquez sur "Dupliquer" ou "Copier"
    • Localiser le bouton d'actions (⋯ ou menu déroulant)
  3. Nommez le nouveau calendrier
    • ID unique : Le système génère automatiquement un suffixe (ex: SUPPORT_CLIENT_N1_COPY)
    • Vous pouvez modifier : Le nom descriptif
  4. Validez
    • Un nouveau calendrier est créé avec l'exact configuration du calendrier source
  5. Modifiez si needed
    • Éditez le nouveau calendrier pour pouvoir le différencier de l'original si nécessaire
Économie de temps : Dupliquer puis ajuster (5 min) est beaucoup plus rapide que créer à zéro (20+ min).

10. Bonnes pratiques et recommandations

✅ À faire

1. Nommer clairement vos calendriers

Utilisez des noms qui décrivent le service, la zone, où le niveau (ex: "SUPPORT_MAIL_TIER2_EUROPE") plutôt que génériques (ex: "CAL1").

2. Définir dans la Semaine Type les règles qui s'appliquent 90% de l'année

Ne mettez en exception que les cas vraiment exceptionnels. Moins vous avez d'exceptions, moins il y a de risques d'erreurs et plus c'est facile à maintenir.

3. Documenter les exceptions

Utilisez toujours le champ "Description" pour les exceptions afin que d'autres utilisateurs comprennent pourquoi cette exception existe.

4. Préférer la duplication

Si vous créez plusieurs calendriers similaires, dupliquez plutôt que de créer à zéro.

5. Valider avant de sauvegarder

Avant de cliquer sur "Sauvegarder", relisez la Semaine Type et les exceptions pour détecter les erreurs.

6. Utiliser le fuseau horaire correct

Si votre équipe est répartie dans plusieurs zones, créez des calendriers séparés avec les bons fuseaux horaires.

❌ À éviter

1. Ne pas confondre exception et Semaine Type

Si vous avez l'habitude de fermer le samedi, c'est une règle Semaine Type, pas une exception. Les exceptions, c'est pour les déviations rares.

2. Utiliser des IDs de calendrier génériques ou mal formatés

Évitez TEMP, TEST, XXXX ou CAL_ABC. Les IDs doivent être auto-explicatifs.

3. Oublier de sauvegarder

Les modifications ne sont pas automatiques. Vous DEVEZ cliquer sur "Sauvegarder" sinon tout est perdu.

4. Créer des exceptions pour chaque jour

Si tous les jours sont en exception, vous vous trompez conceptuellement. Utilisez la Semaine Type pour les horaires normaux.

5. Ne pas tenir à jour régulièrement

Pensez aux événements annuels (vacances d'été, augmentation de volume, maintenance programmée). Préparez les calendriers à l'avance.

11. Erreurs courantes et solutions

Liste des erreurs rencontrées

❌ "Impossible de sauvegarder" ou message d'erreur vague ▼

Causes possibles :

  • Format d'ID invalide : Contient des espaces, accents, ou caractères spéciaux
  • ID déjà existant : Un autre calendrier porte le même ID
  • Horaires invalides : L'heure de fermeture est avant l'ouverture (ex: 18:00 - 09:00)
  • Connexion AWS perdue : Vérifiez votre connexion

Solutions :

  1. Vérifiez que votre ID suit le format LETTRES_CHIFFRES (pas d'espaces)
  2. Vérifiez que les horaires sont au bon format : HH:MM (24h)
  3. Assurez-vous que l'heure d'ouverture < heure de fermeture
  4. Essayez à nouveau après quelques secondes
❌ Les horaires du calendrier ne correspondent pas à la réalité ▼

Causes possibles :

  • Mauvais fuseau horaire : Vous avez sélectionné "UTC" au lieu de "Europe/Paris"
  • Décalage de 1 heure : Confusion entre heure d'été et d'hiver

Solutions :

  1. Ouvrez l'éditeur du calendrier
  2. Cliquez sur Éditer les propriétés
  3. Vérifiez le fuseau horaire : doit correspondre à la localisation de l'équipe
  4. Changez-le si nécessaire
❌ Une exception que j'ai créée n'apparaît pas ▼

Causes possibles :

  • Vous avez oublié de sauvegarder : Vous avez créé l'exception mais pas cliqué sur "Sauvegarder"
  • Le calendrier affiche l'année précédente : La date d'exception est dans 2025, le calendrier affiche 2024

Solutions :

  1. Cliquez sur Sauvegarder
  2. Rafraîchissez le navigateur (F5)
  3. Vérifiez que vous regardez la bonne année
❌ Changement d'heure d'été/hiver : les horaires décalent ▼

Explication :

Ceci est un comportement NORMAL et ATTENDU. Les heures d'été sont gérées automatiquement par Amazon, mais certaines zones peuvent afficher des décalages temporaires.

Solutions :

  1. Vérifiez les dates exactes du changement d'heure (généralement dernier dimanche de mars et octobre)
  2. Créez des exceptions pour les jours autour du changement si nécessaire
  3. Consultez votre administrateur système
❌ Impossible à dupliquer un calendrier ▼

Causes possibles :

  • Permissions insuffisantes : Vous n'avez pas le droit de créer un nouveau calendrier
  • Quota atteint : Nombre maximum de calendriers dépassé

Solutions :

  1. Vérifiez vos permissions avec l'administrateur
  2. Supprimez des calendriers inutilisés si vous avez atteint la limite

12. Questions fréquemment posées (FAQ)

Q: Puis-je modifier un ID de calendrier après sa création ? ▼
R : Non. L'ID est fixe et immuable. Si vous devez absolument changer d'ID, vous devez dupliquer le calendrier avec un nouvel ID et supprimer l'ancien.
Q: Deux calendriers peuvent-ils avoir des horaires qui se chevauchent ? ▼
R : Oui, complètement normal. C'est même courant. Par exemple, un calendrier "Support N1" est ouvert 8h-22h et un calendrier "Support N2" est ouvert 9h-20h. Pendant 9h-20h, les deux files peuvent être actives simultanément.
Q: Comment gérer un service qui fonctionne 24h/24 ? ▼
R : Configurez la Semaine Type ainsi :
  • Tous les jours : Ouvert - 00:00 à 23:59
  • Aucune exception si ces horaires ne changent jamais
Note: Les horaires acceptés vont de 00:00 à 23:59 (fin de journée maximale).
Q: Que se passe-t-il si je supprime un calendrier utilisé par une queue ? ▼
R : Cela généralement génère une erreur ou un dysfonctionnement. Avant de supprimer, assurez-vous que le calendrier n'est assigné à aucun service actif. Consultez votre administrateur ou les configurations Amazon Connect.
Q: Peut-on avoir différents fuseaux horaires pour un même calendrier ? ▼
R : Non. Un calendrier a un seul fuseau horaire. Si vous gérez des équipes dans des zones différentes (ex: Paris et Mumbai), créez des calendriers séparés avec les fuseaux corrects.
Q: À quelle fréquence le calendrier est-il mis à jour dans le système ? ▼
R : Après que vous cliquiez sur "Sauvegarder", les changements sont généralement appliqués immédiatement. Cependant, il peut y avoir un délai de quelques secondes à quelques minutes avant que les changements soient visibles dans Amazon Connect.
Q: Puis-je voir l'historique des modifications d'un calendrier ? ▼
R : Pas directement dans l'interface de gestion des calendriers. Cependant, Amazon Connect et AWS Cloud Trail conservent des logs des modifications. Contactez votre administrateur pour l'accès aux audits.
Q: Comment tester un calendrier avant de l'appliquer en production ? ▼
R : Créez un calendrier de test avec un ID clairement marqué (ex: TEST_SUPPORT). Configurez-le comme vous le souhaitez, puis testez-le en l'assignant à une queue de test avant de l'appliquer aux queues de production.
